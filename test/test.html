How-To

Include the required classes, IDs, and styles

Each AB pair to be tested must

  • Belong to the same class (name is arbitrary), e.g. testAB
  • Have different IDs (again, names are arbitrary), e.g., optionA and optionB
  • Include the default display: none; style (AB.js applies the block attribute to make one option visible)

Putting it all together, an example HTML snippet might look like:

<div class="testAB" id="optionA" style="display: none;">
  <a href="unique URL A">I clicked A!</a>
</div>
<div class="testAB" id="optionB" style="display: none;">
  <a href="unique URL B">I clicked B!</a>
</div>

Determine if you will use the random or cookie implementation

Random

If you select the simple random implementation, then you are nearly done. After sourcing the nanoAB.js file, simply capture your AB test slot(s) and then call the randomSetAB() function.

var debug = true; // set to true or false

var randSlots = ["randomAB"]; // based on the AB test class name
randomSetAB(debug, randSlots);
